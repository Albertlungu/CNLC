**feat**: A new feature for the application or library.

**fix**: A bug fix for an issue in the codebase.

**docs**: Documentation only changes (e.g., to README files or inline comments).

**style**: Changes that do not affect the meaning of the code (whitespace, formatting, missing semicolons, etc.).

**refactor**: A code change that neither fixes a bug nor adds a feature, but improves maintainability or structure.

**perf**: A code change that improves performance.

**test**: Adding missing tests or correcting existing tests.

**chore**: Routine tasks, such as updating build tasks, dependency updates, or other changes that don't fit the other categories.

**build**: Changes that affect the build system or external dependencies.

**ci**: Changes to CI configuration files and scripts.
